Fixed Position

position: fixed is same as background-attachment: property. It is clipped to the browser and its height and width never affect the element. fixd element is called as sticky header. It works only for 1st heading.

z-index: -1; will prioritize which element overlay on other element. Making #fixd z-index:-1 will take it behind the p tag. You will learn further in relative position.

To look nice, Contents of the body should start below #fixd. Body or wrapper div's padding can be increased. But #fixd heading will also move as it is fixed to the page at default place.

Position: Static (default)

Web page render each block level element one after the other in the same order as written in html code in new line. Inline elements are rendered one by one in same line.

Relative Position and z index property:

To move the Heading without affecting the width use postion:relative, left:25px. By default position is position:static; render by order(only one flow) starting left.

P3 - position: relative; left: 25px; top: 25px; no margin

P4 - position: relative; left: 25px; top: -35px;

P5 - position: relative; left: 50px; top: -65px

All 3 elements are overlapped. z-index tells which element should come down and above another. By default, it follow P3, P4, P5 HTML code order.

P6 - position: relative; left: 25px; top: 25px; no margin

P7 - position: relative; left: 25px; top: -35px;

P8 - position: relative; left: 50px; top: -65px

P9 - Element next to P8 which is static position

Relative position is same as static position(one after other as in HTML code) but it allows to move the element top, right, left, bottom relative to its static position( 50px towards left away from its static position). Other Non relative elements P9 will continuosly rendered in its respective static position.
